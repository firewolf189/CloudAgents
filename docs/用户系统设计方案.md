# 用户系统设计方案

## 背景

当前系统使用 `X-User-ID` header 进行用户标识，没有任何鉴权，任何人填一个字符串就能登录并创建无限 agent。需要改为**部门级管控**模式：管理员统一管理员工和 agent 分配。

## 数据模型

```
部门（隐含，就是这一套部署）
├── 管理员（环境变量配置，唯一）
├── 凭证（credential）— 部门级，所有人共享
├── 员工列表（Redis）
│   ├── 员工 A（name, token）
│   │   ├── Agent 1（管理员创建并分配）
│   │   └── Agent 2
│   └── 员工 B（name, token）
│       └── Agent 3
```

## 角色与权限

| 能力 | 管理员 | 员工 |
|------|--------|------|
| 登录方式 | 用户名 + 密码 | Token |
| 查看 Agent | 所有 | 仅分配给自己的 |
| 创建/编辑/删除 Agent | ✅ | ❌ |
| 分配 Agent 给员工 | ✅ | ❌ |
| 管理凭证（Credential） | ✅ | ❌（不感知） |
| 员工管理（创建/禁用/发 token） | ✅ | ❌ |
| 聊天对话 | ✅ | ✅ |

## 认证流程

- **管理员**：用户名 + 密码登录 → 返回 JWT/session
- **员工**：粘贴 token 登录 → 后端校验 token 有效 → 返回身份信息
- 所有 API 请求带 `Authorization: Bearer <jwt/token>` 替代当前的 `X-User-ID`
- 管理员账号密码通过环境变量或配置文件写死（唯一管理员）
- 员工 token 由管理员生成，永久有效直到管理员撤销

## 前端变化

### 登录页

- 两种登录方式切换："管理员登录"（用户名 + 密码）/ "员工登录"（Token）

### 管理员视角

- 新增「员工管理」页面（左侧导航加一个 tab）：
  - 创建员工、生成 token
  - 查看/复制 token
  - 禁用/删除员工
- Agent 列表显示所有 agent + 归属员工名
- 创建 agent 时多一个「分配给」下拉选择员工

### 员工视角

- 看不到凭证页、员工管理页
- Agent 列表只显示分配给自己的
- 其他不变（正常聊天、选模型等）

## 后端变化

### 认证中间件

替换 `X-User-ID` header 解析，改为 token 校验，注入用户身份（user_id, role）

### 新增员工管理 API

- `POST /users/` — 创建员工（管理员专用）
- `GET /users/` — 列出所有员工（管理员专用）
- `DELETE /users/{user_id}` — 删除/禁用员工（管理员专用）
- `POST /users/{user_id}/token` — 重新生成 token（管理员专用）

### Agent 分配

- Agent 记录新增 `assigned_to` 字段（员工 user_id）
- 创建 agent 时指定 `assigned_to`

### 权限拦截

- 员工只能访问 `assigned_to` 等于自己的 agent 及其 session
- 凭证相关 API 员工不可访问

### 凭证隔离

- 凭证存储在管理员 namespace 下
- 员工查模型列表时透传管理员的凭证

## 不动的部分

- 现有 Redis 数据结构不迁移
- Session / Message / Workspace 逻辑不变
- MCP / Skill 机制不变

## 落地顺序

1. 后端：认证中间件 + 管理员登录 API
2. 后端：员工管理 CRUD API
3. 后端：Agent `assigned_to` 字段 + 权限拦截
4. 前端：登录页改造（管理员/员工双模式）
5. 前端：员工管理页面
6. 前端：Agent 创建时分配员工
7. 前端：员工视角权限隔离（隐藏凭证页��）
